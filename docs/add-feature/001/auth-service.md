# Auth Service の変更内容

## 必要な変更
- ポイント管理サービスが認証を行うために、既存のトークン検証エンドポイント (`/auth-service/api/auth/verify`) を利用。
- 必要に応じて、スコープや権限の追加を検討。

## JWT構造の確認と修正タスク
- **要件**: JWTトークンにはユーザーID（UUID形式）が含まれる必要があります。
- **現状確認**: 現在のJWTトークンにユーザーIDが含まれているか確認が必要。
- **修正タスク**: もしJWTトークンにユーザーIDが含まれていない場合、以下の修正が必要：
  - JWTのペイロードに `userId` クレーム（UUID形式）を追加。
  - トークン生成ロジックを更新。
  - トークン検証エンドポイントがユーザーIDを返却するように修正。
